  1. Gather to get this in long format using tidyr::gather (note: or use pivot_longer from the development version of tidyr using devtools::install_github(hadley/tidyr) - they do the same thing, but the pivot_longer syntax is more intuitive than gather).

Here: what is tidy format? Each variable is a column; each row is an observation. Here, we have one variable (time) spread out over many columns (wide format). This is not tidy data - lets convert by compressing all year information into a single column, with the values attached.

biofuels_gather <- biofuels %>% 
  gather(key = "year", value = "biofuel_amount", '2000':'2010')

# Use head(biofuels_long) in the console to see the first 5 lines, ensuring that it's been gathered (or View() to see the whole df). Also notice that the class of 'year' is still a character (since it's been converted from a column name, which is always a character). Let's change the class of the year column to numeric (dbl can integer or numeric). 

biofuels_gather$year <- as.numeric(biofuels_gather$year)

# Now check the class again (you can use class(biofuels_long$year) in the console)

The World Bank data is similarly in wide format (years spread across columns). Lets gather that, too:

OK, now lets clean up column names for each

  1. janitor::clean_names()

Clean up the names using janitor::clean_names (default is snake_case)

biofuels_2 <- clean_names(biofuels_gather)
wb_data_2 <- clean_names(wb_gather)

Use names to see what the new column names are. Are they similar across datasets for the same variables (e.g.year and year, country and country, etc.)?

Here, we see that in biofuels_2 the country name is in a column called id, but in wb_data_2 it is stored in column country_name. We need those column names to match so that we can join our datasets.

  1. dplyr::rename()

Rename the column containing country names so that they match across data frames we want to join. Here, Ill rename id to country_name in the biofuels_2 df:

biofuels_rn <- biofuels_2 %>% 
  rename(country_name = id)

Great. Now both data frames have a column country_name and a column year - and when we join them together, R will look for matching column names, and matching levels within columns, to merge the data together.

  1. Join data frames by matching column names
df_join <- full_join(biofuels_rn, wb_data_2)
## Joining, by = c("country_name", "year")

Notice that it tells you which variables its joining by.

There are other options for joins, but full_join is kind of the safest (all data is retained, even if there are matches in both datasets).

So now we have all of our data together. Lets do some additional wrangling, analysis and visualization.

  1. Wrangling (select, filter)

Lets say that we only want to retain information for the following:

  • Total greenhouse gas emissions (kt of CO2 equivalent), EN.ATM.GHGT.KT.CE
  • From 2000 - 2010
  • For countries: United States, China, India, Russia
df_subset <- df_join %>% 
  filter(year >= 2000 & year <= 2010) %>% 
  filter(indicator_code == "EN.ATM.GHGT.KT.CE") %>% 
  rename(tot_ghg = indicator_value) %>% 
  filter(country_name %in% c("United States","China","India")) %>% 
  arrange(country_name)
  1. dplyr::group_by + summarize - a powerful combo

Lets say we want to calculate the total greenhouse gas emissions (kt CO2 equivalents) and biofuels consumption (thousands barrels per day) for each country from 2000 - 2010. Well use group_by + summarize:

totals <- df_subset %>% 
  group_by(country_name) %>% 
  summarize(
    ghg = sum(tot_ghg),
    biofuel = sum(biofuel_amount)*365*10*1000
  )

in the real world Id convert units, but Im fine with the mismatch here.

  1. In-line variable referencing
  • Why is this so cool? Because it means as your data/code are updated, so will be your document. That is a) awesome, b) efficient, c) reproducible, d) way less risky than re-copying & pasting values/outcomes whenever your data or analyses are updated.

Reference variables from code using two backticks with an r in between, then whatever you want to do.

Like this: 853.703

So it doesnt look like a value in code, but when you knit then the actual value shows up. Very cool.

You can also create variables within a code chunk, then just reference the variable name. For example, lets create a variable that calculates the total GHG emissions from all three countries in df_subset from 2000 - 2010.

ghg_sum <- sum(df_subset$tot_ghg)

  1. group_by_split() if time.

What if we want to work with emissions/biofuels for each country separately? Well, one option is to create subsets using filter to match country nameswhich isnt bad if you have only three (like we do), but what if there are 100? Then we might want an easier way. dplyr::group_split() provides it.

df_split <- df_subset %>% 
  group_by(country_name) %>% 
  group_split()

Now, this seems like a kind of weird format if you just look at df_split, e.g.in the console since its multiple tibbles stored within a list. We just need to access them by calling the appropriate index.

Check out df_split in the console. Notice that there is [[1]] for China, [[2]] for India, and [[3]] for United States - well use these indices to get the tibble (a slightly more advanced & functional data frame, but you can think of them interchangeably). Like, to get just a data frame for the US data:

us_df <- data.frame(df_split[3])
